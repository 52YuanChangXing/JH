# 简浩影视 · Jianhao Film Studio

一个融合前端展示与后端管理的摄影工作室网站，包含作品展示、团队介绍、预约表单、进度追踪看板等功能模块。项目使用原生 Node.js 构建 API 与静态资源服务，无需额外依赖即可运行。

## 功能亮点

- 📸 **作品集与分类筛选**：展示婚礼、旅拍、商业等核心案例，支持按类型过滤。
- 👥 **摄影导演团队**：展示摄影师履历、擅长领域与代表案例。
- 🗓️ **智能预约表单**：收集客户信息并在后端持久化保存，自动归档到看板。
- 📊 **项目进度看板**：以策划、拍摄、后期、交付四大阶段展示每个项目的实时状态。
- 💬 **客户推荐与 FAQ**：提升品牌可信度并解答常见问题。
- ☎️ **多渠道联系**：电话、邮箱、微信、社交媒体等一应俱全。
- 🛠️ **后台管理面板**：通过 `/admin` 登录后，可在线编辑英雄区文案、功能开关、作品/服务列表，以及同步预约进度，直接驱动前端组件展示。
- 🔁 **前后台实时联动**：内容、预约、进度更新通过事件流推送至前台与后台预览，实时验证改动效果。

## 快速开始

```bash
# 进入项目目录
cd server

# 启动服务（默认端口 8080）
npm start
```

服务启动后，可访问 `http://localhost:8080` 浏览主站，访问 `http://localhost:8080/admin` 进入后台。默认管理令牌为 `jianhao-admin-secret`，可通过环境变量 `ADMIN_TOKEN` 自定义。

所有接口位于 `/api` 前缀下，例如：

- `GET /api/content`：基础文案与静态内容
- `GET /api/portfolio`：作品集数据
- `GET /api/bookings`：预约记录列表
- `POST /api/bookings`：提交新的预约申请
- `PATCH /api/bookings/:id/progress`：追加进度节点（需在请求头携带 `X-Admin-Token`）

数据默认存储在 `server/data` 目录下的 JSON 文件中，可根据需要自行扩展或接入真实数据库。

## 后台管理功能

- **内容编辑**：在“首页英雄区”“品牌指标”等面板即时修改文案，自动写回 `static-content.json`，刷新主站即可生效。
- **组件开关**：通过勾选功能开关，决定前端是否展示作品、服务、FAQ 等模块，同时更新导航链接可见性。
- **集合维护**：以 JSON 数组的方式批量管理作品、服务、摄影师等列表，也支持新增、删除单条记录。
- **整站同步**：提供完整的内容 JSON 编辑器，方便从外部系统导入或批量调整站点结构。
- **预约运营**：后台可以查看预约详情、历史进度并追加新的阶段节点，前端客户看板即时同步。
- **前台预览**：后台内置 `/` 站点的实时 iframe 预览，支持手动刷新并在内容更新后自动同步效果。

## 项目结构

```
JH/
├── README.md          # 项目说明
└── server/
    ├── index.js       # Node.js 服务入口
    ├── package.json   # npm 配置（仅依赖内置模块）
    ├── data/          # 静态内容与预约数据
    └── public/        # 前端页面、样式、脚本与图像资源
```

## 自定义与扩展建议

- 将 `data/static-content.json` 替换为真实的 CMS 或数据库输出。
- 在 `data/bookings.json` 中扩展字段，例如费用、人员配置等。
- 为 `/api/bookings/:id/progress` 提供权限校验或后台管理界面。
- 使用 Nginx 等反向代理为静态站点启用 HTTPS 与缓存策略。
